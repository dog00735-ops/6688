# 輿情監控說明書

## 1. 這支程式現在在做什麼

`新聞.py` 已經從原本偏政治新聞快報，整理成「台灣輿情監控」版本。
它會：

- 從多個 RSS 來源抓新聞
- 用關鍵字篩出和你監控主題有關的內容
- 做基本分類，例如民調聲量、選戰動態、國會攻防、兩岸外交、政策議題、爭議事件
- 可選擇用 OpenAI 幫你做摘要與風向判定
- 寫進 SQLite 資料庫
- 可推送 Telegram 快報與日報

## 2. 資料夾內主要檔案

- `新聞.py`：主程式
- `.env`：你的正式參數設定
- `.env.example`：參數範例
- `cron.example`：排程範例
- `political_intel.db`：資料庫主檔
- `political_intel.log`：執行紀錄
- `political_intel.db-wal`、`political_intel.db-shm`：SQLite 正常運作時的附屬檔

## 3. 最常改的參數

請修改 `.env`：

- `MONITOR_NAME`
  用來改 Telegram 標題，例如 `台灣輿情監控`、`選戰輿情雷達`

- `MONITOR_KEYWORDS`
  逗號分隔的關鍵字清單，程式會用它決定哪些新聞要留下

- `PRIORITY_KEYWORDS`
  高優先關鍵字，命中時重要度會更高

- `MAX_ITEMS_PER_RUN`
  每次最多抓幾則候選新聞

- `DAILY_REPORT_LIMIT`
  日報最多列幾則

- `DEDUPE_HOURS`
  幾小時內視為重複或相似內容

- `ENABLE_TELEGRAM`
  `true` 或 `false`，控制是否推送 Telegram

- `OPENAI_MODEL`
  要使用的模型名稱

## 4. 進階可調參數

- `POSITIVE_HINTS`
  影響風向判斷的正向詞

- `NEGATIVE_HINTS`
  影響風向判斷的負向詞

- `ENTITY_HINTS_JSON`
  自訂人物、政黨、機構判斷

- `TOPIC_HINTS_JSON`
  自訂主題分類規則

範例：

```env
ENTITY_HINTS_JSON={"民進黨":["民進黨","賴清德"],"國民黨":["國民黨","侯友宜"]}
TOPIC_HINTS_JSON={"民調聲量":["民調","支持度"],"爭議事件":["弊案","起訴"]}
```

## 5. 怎麼手動執行

只列出本次抓到的候選：

```bash
python3 '/Users/wangpindun/Library/Mobile Documents/com~apple~CloudDocs/python_project/新聞.py' --list-only
```

正式監控：

```bash
python3 '/Users/wangpindun/Library/Mobile Documents/com~apple~CloudDocs/python_project/新聞.py' --mode monitor
```

產生日報：

```bash
python3 '/Users/wangpindun/Library/Mobile Documents/com~apple~CloudDocs/python_project/新聞.py' --mode daily-report --hours 24 --limit 12
```

## 6. 怎麼改排程

請修改 `cron.example`，改完後重新安裝：

```bash
crontab '/Users/wangpindun/Library/Mobile Documents/com~apple~CloudDocs/python_project/cron.example'
```

查看目前排程：

```bash
crontab -l
```

## 7. 常見問題

### Finder 看不到 `.env`

macOS 按 `Command + Shift + .` 顯示隱藏檔。

### 為什麼會有 `.db-wal` 和 `.db-shm`

這是 SQLite 的正常附屬檔，不是壞掉，也不要手動刪除。

### 改完 `.env` 要重開機嗎

不用。下次程式執行時就會讀到新設定。

## 8. 建議你怎麼改

如果你想把它變成不同用途，可以優先改這三個：

1. `MONITOR_NAME`
2. `MONITOR_KEYWORDS`
3. `PRIORITY_KEYWORDS`

這樣就可以很快從政治輿情，改成你想看的主題監控。
